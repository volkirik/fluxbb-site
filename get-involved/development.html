FluxBB on GitHub Follow development, get the source, test the code, and contribute

FluxBB uses GitHub to maintain its development repositories, allowing you to follow development and download the latest source code to test and report issues.

Visit FluxBB on GitHub ->

Development Discussion Community contributors can participate in the planning FluxBB's future

FluxBB maintains a set of Development forums on the FluxBB Community Forums to allow all FluxBB users to report bugs and issues with their forums, make suggestions for features to be included in future releases, and view discussions related to FluxBB development. Additionally, active community members can get involved with development discussions by joining a special Contributors group on the FluxBB Community Forums.

Development discussions on the FluxBB Community Forums ->

Reporting Bugs and Issues Help FluxBB find and resolve issues with the software

When you experience a problem with your FluxBB forum, it's important you report the issue so the FluxBB team can investigate the issue and fix the issue if necessary. Bug reports can be created using your FluxBB Community account in the bug reports forum for the appropriate version of FluxBB within the Development category on the FluxBB Community Forums.

More information on reporting bugs and issues ->

Development Workflow FluxBB uses a workflow to manage how development occurs on GitHub

To help streamline the coordination of development between FluxBB team members and members of the FluxBB community, FluxBB follows a documented workflow for the development of all of its products, including FluxBB 1.4, 1.5, 2.0, and the FluxBB Merge System.
